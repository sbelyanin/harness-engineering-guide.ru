# Сравнение основных реализаций Harness

Это фактическое сравнение заметных agent-harness по состоянию на середину 2025 года. Область развивается быстро — конкретные детали могут меняться, но архитектурные паттерны остаются стабильными.

|---|---|---|---|---|---|
| **OpenClaw** | Инжекция context на основе AGENTS.md. Загружает файлы workspace'а, skills и memory на каждую сессию. Сжатие context через саммаризацию. | Файловая: MEMORY.md (долгосрочная), дневные логи (memory/YYYY-MM-DD.md), wiki/ для структурированных знаний. Оркестратор делегирует задачи, результаты анонсируются автоматически. | ✅ Open source |

## Углублённо: ключевые отличия

### Стратегия Context

Как каждый harness решает, что показывать модели:

```
Aider          : Явная — пользователь добавляет файлы вручную (/add, /drop)
Claude Code    : По требованию — агент читает файлы по мере необходимости
Cursor         : Семантическая — индекс embeddings, авто-поиск релевантного кода
OpenClaw/Nexu  : Конфиг-управляемая — AGENTS.md декларирует, что загружать
Codex          : Repo-aware — сканирует структуру, загружает релевантные файлы
Cline          : IDE-aware — открытые табы + файлы workspace
```

**Трейд-офф:** Явный контроль (Aider) даёт точность, но требует усилий пользователя. Семантический поиск (Cursor) автоматический, но может пропустить или включить не те файлы. Конфиг-управляемый подход (OpenClaw) предсказуем, но требует предварительной настройки.

### Архитектура Memory

```python
# Type 1: Нет memory (Claude Code, Codex, Cursor, Cline)
# Каждая сессия начинается с чистого листа. Context берётся из файлов проекта.
context = load_project_files()  # И всё

# Type 2: Файловая memory (OpenClaw, Nexu)
# Персистентное знание между сессиями, редактируемое пользователем.
context = (
    load_project_files()
    + load_memory("MEMORY.md")
    + load_daily_log(today)
    + load_wiki_if_relevant(task)
)

# Type 3: Встроенная memory (проприетарные платформы)
# Хранится в векторной БД, извлекается по сходству.
context = (
    load_project_files()
    + vector_search(task, memory_store)
)
```

### Расширяемость Tools

| Подход | Проекты | Плюсы | Минусы |
|---|---|---|---|
| **MCP Protocol** | OpenClaw, Claude Code, Cline | Стандартный протокол, интероперабельность | Оверхед настройки серверов |
| **Skill Files** | OpenClaw, Nexu | Самодостаточные, шеребельные, маркетплейс | Кастомный формат |
| **Built-in Only** | Aider, Codex | Просто, предсказуемо | Ограниченная расширяемость |
| **IDE Integration** | Cursor, Cline | Богатый editor context | Привязка к IDE |

### Паттерны Multi-Agent

Большинство harness — single-agent. Поддержка multi-agent сильно варьируется:

```python
# OpenClaw/Nexu: Первоклассное порождение sub-agent'ов
subagent = spawn(
    task="Research competitor pricing",
    model="gpt-4o",
    tools=["web_search", "web_fetch"],
)
# Результат авто-анонсируется по завершении — polling не нужен

# Claude Code: Косвенный multi-agent через exec
result = exec("claude-code --print 'Review this PR'")
# Работает, но нет структурированной коммуникации

# Остальные: Нет нативного multi-agent
# Workaround: несколько терминальных сессий, ручная координация
```

## Как выбрать Harness

| Если вам нужно... | Рассмотрите |
|---|---|
| Глубокое редактирование кода с IDE-интеграцией | **Cursor** или **Cline** |
| Terminal-based coding agent | **Claude Code** или **Aider** |
| Sandboxed-выполнение задач | **Codex** |
| Multi-agent-оркестрация | **OpenClaw** или **Nexu** |
| Персистентная memory между сессиями | **OpenClaw** или **Nexu** |
| Расширяемая экосистема tools | **OpenClaw** (skills + MCP) |
| Минимум настройки, просто работает | **Aider** |

## Чего таблица не показывает

- **Поддержка моделей** — Большинство harness модельно-агностичны (OpenClaw, Cline, Aider поддерживают много провайдеров). Codex — только OpenAI. Claude Code — ориентирован на Anthropic.
- **Стоимость** — Сильно варьируется в зависимости от выбора модели, размера context и сложности задачи. Это не feature harness, а паттерн использования.
- **Скорость** — Зависит больше от латентности модели, чем от архитектуры harness. Поддержка streaming универсальна.
- **Сообщество** — У open-source проектов (Aider, Claude Code, Cline, OpenClaw) активные сообщества. Размер сообщества ≠ качество.

## Частые ловушки

- **Выбор по хайпу** — Берите harness под свой workflow, а не тот, у которого больше звёзд на GitHub.
- **Игнорирование memory** — Если ваши задачи растягиваются на несколько сессий, harness без персистентной memory означает переобъяснение context каждый раз.
- **Считать, что «open source» = «бесплатно»** — Сам harness бесплатен; API-вызовы модели — нет. Закладывайте бюджет на API-затраты.
