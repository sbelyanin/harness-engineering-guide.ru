# Cyrillic-Log-Analyzer — разбор логов с битой кодировкой и транслитом

Превращает сырой лог (или любой текст) с типично-российскими артефактами кодировки/раскладки в:
1. **Отчёт** — сколько строк «болеет», чем именно, и как чинить.
2. **Исправленный текст** — авто-fix mojibake и переключение раскладки.

Без внешних зависимостей — только Python 3.10 stdlib.

## Когда использовать

- **Prod-логи от legacy-сервиса** — старая Java/C++ на Windows отдаёт cp1251, pipeline ждёт UTF-8, в Kibana видим «РџСЂРёРІРµС‚»
- **Миграция данных** из старых СУБД (MS SQL с collation Cyrillic_General_CI_AS, DBF, 1C 7.x выгрузки)
- **Пользовательский ввод** — комментарии, тикеты, обращения, где половина в транслите или «набрано не переключив раскладку»
- **Подготовка датасетов** — перед eval/training убедиться, что mojibake не попал в размеченные данные
- **Разбор тредов из мессенджеров** — смесь кириллицы, транслита, смайликов, parasing из Latin-1
- **Forensic** — анализ crash-dump и stack-trace из Windows-приложений

## Что считается проблемой

Skill различает **6 категорий** строк:

| Категория | Признак | Пример |
|-----------|---------|--------|
| `clean_cyr` | >85% букв — кириллица UTF-8 | `Привет, мир!` |
| `clean_lat` | >85% букв — латиница, похоже на английский | `INFO Service started` |
| `mixed` | Кириллица + латиница в одной строке | `INFO Привет world` |
| `mojibake` | UTF-8 прочитан в чужой кодировке | `РџСЂРёРІРµС‚` или `ÐÑÐ¸Ð²ÐµÑ` |
| `translit_likely` | Латиница с типично-русскими диграфами | `Privet mir, ya lyublyu` |
| `empty` | Нет букв | `---` или пустая строка |

## Процесс анализа (5 шагов)

### Шаг 1: Классификация каждой строки

Для каждой непустой строки лога:

```python
cls = classify_line(line)
# cls.status  — clean_cyr | clean_lat | mixed | mojibake | translit_likely | empty
# cls.detail  — utf8→cp1251 / utf8→latin1 / english / unknown-lang / cyr=XX% lat=YY%
# cls.fixed   — восстановленный текст для mojibake (иначе None)
```

### Шаг 2: Детекция mojibake

Mojibake — самая частая и самая «дорогая» проблема (данные теряются). Skill пытается round-trip:

```python
# UTF-8 прочитанный как CP1251 — lead bytes 0xD0/0xD1 становятся 'Р'/'С'
try_fix_cp1251_mojibake(text)   # text.encode('cp1251').decode('utf-8')

# UTF-8 прочитанный как Latin-1 — lead bytes 0xC3 0x90/0x91 становятся 'Ð'/'Ñ'
try_fix_latin1_mojibake(text)   # text.encode('latin-1').decode('utf-8')
```

Валидация: фиксированный текст должен содержать **больше строчной кириллицы** (а-я), чем исходный. В mojibake строчной кириллицы почти нет — там uppercase Р/С/Ð/Ñ как lead bytes.

**Ложные срабатывания** возможны, если в исходном тексте много латинских слов с `Ð`/`Ñ` (редкость). round-trip либо даёт осмысленный русский, либо падает с UnicodeError — третий исход маловероятен.

### Шаг 3: Детекция транслита

Эвристика (без словаря — deterministic):

- Латиница > 85% букв
- Присутствует хотя бы один типично-русский диграф: `sh, ch, zh, ya, yu, yo, kh, ts, shch, aj, oj, uj, ej, ij, yj, ja, ju, jo`
- Или окончания `-yj`, `-ij`, `-aja`, `-oje`, `-ech`, `-ck`
- И при этом **нет** английских стоп-слов (the, and, for, with, ...)

Транслит **не детерминирован** — одно слово можно записать 3-5 способами. Skill только помечает строку как подозрительную и предлагает конвертацию с предпросмотром.

### Шаг 4: Транслитерация (опционально)

`cmd_translit` конвертирует транслит в кириллицу. Три схемы:

| scheme | Что делает | Пример |
|--------|------------|--------|
| `auto` (default) | Диграфы + однобуквенные | `Privet` → `Привет`, цифры сохраняются |
| `volapuk` | + `4→ч`, `6→ш`, `^→щ` (хакерский 90-х) | `4to delaesh` → `что делаеш` |
| `gost` | Псевдоним `auto` | для SSOT по имени |

**Неоднозначные буквы** (помечаются в отчёте):

- `w` — `в` (ГОСТ / Scientific) или `ш` (английский-like)
- `y` — `ы` (по умолчанию), `й` (в дифтонгах `aj/oj`), `и` (ICAO Doc 9303)
- `x` — `х` или `кс`
- `j` — `й` (научная) или `ж` (французская)
- `c` — `ц` (по умолчанию) или `к` (перед e/i: `ce→ке`)
- `h` — `х` (если не часть диграфа `sh/ch/zh`)

**Всегда** делайте предпросмотр конвертации перед массовым применением.

### Шаг 5: Переключение раскладки (опционально)

`cmd_layout` переключает qwerty↔йцук для случая «забыл переключить раскладку»:

```bash
echo "ghbdtn" | python analyze.py layout     # → привет
echo "привет" | python analyze.py layout --direction cyr→lat   # → ghbdtn
```

В mixed-строках переключает **весь текст** — для per-word анализа нужен отдельный pass (см. Антипаттерны).

## Композитная оценка лога

Лог оценивается 0–12 баллами (3 оси × 0–4 + бонус):

| Ось | 0 | 4 |
|-----|---|---|
| % mojibake | 0% | ≥20% |
| % транслита | 0% | ≥40% |
| % смешанных | 0% | ≥40% |
| Бонус +2 | — | несколько типов mojibake одновременно |

**Интерпретация:**
- 0–4: ✅ лог чистый, вмешательство не требуется
- 5–9: ⚠️ есть проблемы — стоит прогнать `fix`
- 10–12: 🚨 критический — данные теряются, нужен срочный анализ pipeline

## Формат вывода

```markdown
# Cyrillic-Log-Analyzer Report

## Источник
- Файл: `app.log`
- Строк всего: 12453 (непустых: 11820)

## Сводка
- Mojibake: **412** (3.5%)
- Транслит (вероятный): **87** (0.7%)
- Смешанные строки: **2104** (17.8%)
- Чистая кириллица: 6980
- Чистая латиница: 2237
- Общий балл: **6 / 12** — ⚠️ есть проблемы — стоит прогнать fix

## Типы mojibake
| Тип | Строк |
|-----|-------|
| utf8→cp1251 | 388 |
| utf8→latin1 | 24  |

## Неоднозначные буквы в транслите (top-5)
| Буква | Встречаемость |
|-------|---------------|
| y | 142 |
| j | 38 |
...

## Примеры
### mojibake
- L482: `WARN Пользователь РџС‚РµС‚РµСЂРёРЅ...`
  → `WARN Пользователь Птетерин...`
### translit_likely
- L1023: `INFO Ya lyublyu ispol'zovat' Python`
...

## Рекомендации
- Запустите `fix app.log > app.fixed.log` — автоматически восстановит UTF-8.
- Для транслита прогоните `translit --scheme volapuk` (предпросмотр обязателен — схема неоднозначна).
```

## Команды

```bash
# Анализ (stdin или файл)
python analyze.py analyze app.log
cat *.log | python analyze.py analyze -

# Авто-fix mojibake (вывод на stdout)
python analyze.py fix app.log > app.fixed.log

# Транслит → кириллица
echo "Privet mir" | python analyze.py translit --scheme volapuk

# Переключение раскладки
echo "ghbdtn" | python analyze.py layout
echo "привет" | python analyze.py layout --direction cyr→lat
```

## Адаптация

- **Очень большой лог (>100MB):** `analyze` читает потоково, без загрузки в память. Для `fix` тоже — output идёт на stdout построчно.
- **Mixed-encoding в одной строке:** skill не пытается чинить — помечает как `mixed` и рекомендует ручной разбор.
- **Крипто-логи / бинарный мусор:** будут классифицированы как `empty` (нет alpha) или `clean_lat` (неанализируемая латиница).
- **Не-русский кириллический текст** (украинский, белорусский, казахский на кириллице): детектор mojibake работает (UTF-8 ↔ CP1251 универсальна для кириллицы), но транслит-эвристики заточены под русские диграфы.

## Антипаттерны

- **Массово применять `translit` без предпросмотра** — неоднозначные буквы (`w`, `y`, `j`, `c`) дадут разный результат в разных схемах. Всегда сэмплируйте 100-200 строк и сверьте с ожидаемым.
- **Применять `layout` к mixed-строкам целиком** — переключит весь текст, включая английские технические термины. Сначала сегментируйте по словам.
- **Доверять авто-fix'у mojibake на 100%** — round-trip через CP1251 иногда даёт false negative для строк, где русский текст смешан с немецким umlaut или французским accent. Валидируйте выборочно.
- **Игнорировать `mixed`-строки** — если их >30%, скорее всего в лог слиплись разные источники (например, stdout + stderr). Разделите по источнику перед анализом.
- **Конвертировать транслит в production-датасет без ревёрс-проверки** — сэмпл перед обучением модели должен пройти human review.
