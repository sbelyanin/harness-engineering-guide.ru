# Контрибьюшен в Harness Engineering Guide (RU)

Спасибо за интерес к проекту! Этот гайд — общественная инициатива, и мы рады любым формам участия.

## Как можно помочь

### 📝 Контент
- **Разделы гайда** — расширяйте существующие темы или добавляйте новые
- **Туториалы** — пишите пошаговые руководства с примерами кода
- **Обзоры инструментов** — добавляйте обзоры harness-инструментов и фреймворков
- **Конспекты статей** — пересказывайте релевантные академические работы

### 📰 Еженедельный дайджест
- Присылайте интересные ссылки, статьи или инструменты, которые вы нашли
- Откройте issue с тегом `weekly-submission`

### 🌐 Переводы
- Мы ориентируемся на русскоязычное издание; приветствуются правки и улучшения перевода
- Единообразие терминов — см. `ROADMAP.md` (раздел «Конвенция перевода»)

### 🇷🇺 RU-оригинальные статьи

Помимо переводов из upstream, издание активно расширяется **собственными статьями** под русскоязычную аудиторию. Тематика приоритета (см. Track B в `ROADMAP.md`):

- **On-prem / air-gapped harness** — локальный inference (vLLM, Ollama), vending tools, подписанные bundle’ы
- **Российские модели** — интеграция YandexGPT / GigaChat / GigaChat MAX как провайдеров
- **Open-source LLM-стек** — vLLM + локальные модели как замена API-провайдерам
- **Токен-экономика русского текста** — оптимизация под кириллицу
- **Compliance** — 152-ФЗ, аудит логов, обработка ПДн в harness
- **Кейсы RU-команд** — реальные истории внедрения

**Эталон по структуре** — `guide/russian-llm-harness.md` и `guide/on-prem-harness.md`. Перед написанием новой статьи откройте Issue с темой, чтобы согласовать подход.

### 🧰 RU-оригинальные skills

Готовые skills под русскоязычные задачи приветствуются. Структура:

```
skills/<skill-name>/
├── SKILL.md          # frontmatter (name, description) + процесс
└── scripts/          # опциональные утилиты (Python/bash)
```

Эталон — `skills/ru-doc-summarizer/` (саммаризация ГОСТов и регламентов). См. Track C в `ROADMAP.md`.

### 🐛 Исправления
- Опечатки, битые ссылки, устаревшая информация — всё приветствуется

## Как отправить изменение

1. Сделайте форк репозитория
2. Создайте ветку: `git checkout -b your-topic`
3. Внесите изменения
4. Откройте Pull Request с понятным описанием

## Стиль изложения

- **Будьте практичны** — сопровождайте текст примерами кода и реальными сценариями
- **Будьте объективны** — сравнивая инструменты, честно показывайте компромиссы
- **Будьте кратки** — цените время читателя
- **Указывайте источники** — ссылайтесь на оригинальные статьи, посты и репозитории

## Конвенция перевода

Технические термины (Harness, Agent, Context, Tool, Sandbox, Guardrails и т.д.) остаются на английском. Переводится только связующая проза. Подробности — в [`ROADMAP.md`](ROADMAP.md).

## Кодекс поведения

Будьте взаимовежливы, конструктивны и открыты. Мы все здесь, чтобы учиться.
