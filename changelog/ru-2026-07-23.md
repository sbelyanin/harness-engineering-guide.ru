# 2026-07-23 — Завершение Этапа 8

Большой релиз: сайт превратился из функционального перевода в зрелый продукт. Четыре трека работали параллельно — UX чтения, защита от регрессий, SEO-видимость, углубление ключевых статей.

## Track E — Site features (UX чтения)

- **E1. Reading-time** в каждой статье: `computeReadingTime()` в `content.ts`, ~150 wpm, вычитает code-блоки (не считает «время чтения» JSON-примера в 200 строк).
- **E2. «Связанные статьи»** внизу каждой статьи: `getRelatedArticles()` в `guide-data.ts`. Алгоритм скоринга: +5 за ту же `section`, +1 за общий токен в title. Показывает 3 лучших.
- **E3. RSS-фид `/feed.xml`**: RSS 2.0 + Atom self-link. Только `ru-` prefixed changelog entries (отделяет собственные изменения RU-издания от переводов upstream). Autodiscovery `<link>` в `<head>`.
- **E4. Client-side поиск**: `/search.json` (static, ~200 KB на 31 статью) + `SearchDialog` (zero-dep fuzzy substring). Алгоритм: title ×10, description ×5, body ×1, бонус ×2 за покрытие всеми query-terms. Подсветка совпадений, keyboard nav ↑↓/Enter/Esc, shortcut `/` или `Cmd/Ctrl+K`.

## Track F — Quality gates (защита от регрессий)

Четыре Python-чекера (stdlib-only, без внешних зависимостей — иначе ломается deploy) + CI-интеграция.

- **F1. `check_frontmatter.py`** — `title` + `section` + `author` обязательны в `guide/*.md`; `section` ∈ enum (`getting-started`, `core-concepts`, `practice`, `reference`, `showcase`).
- **F2. `check_registry.py`** — детект drift'а между `guide/*.md` и `guide-data.ts`. Новый markdown-файл без регистрации → не попадёт в sidebar (частый баг при добавлении статей).
- **F3. `check_links.py`** — внутренние ссылки резолвятся: `/guide/<slug>`, относительные пути, README/ROADMAP cross-links. CommonMark-совместимый парсер code-fences (nested fences через ≥4 backticks).
- **F4. `check_style.py`** — STYLE.md linter: ASCII `"..."` в prose, русские транслитерации EN терминов (`фреймворк`/`пайплайн`/...), NBSP в исходниках.
- **F5. `.github/workflows/quality.yml`** — F1–F4 на каждый PR и push в `main`.

## Track G — SEO & discovery

- **G1. Sitemap** расширен: добавлен `/community`, `lastModified` из `frontmatter.date` или `fs.mtime`, приоритеты + changeFrequency, anchor-ссылки для changelog-записей.
- **G2. JSON-LD `Article` schema** на каждой guide-странице: `<script type="application/ld+json">` с `headline`, `datePublished`, `author`, `description`. Учитывается Google и Yandex.
- **G3. OG-изображения**: `banner.png` подключён в `layout.tsx` (site-wide default) и в guide-страницах (`openGraph.images` + `twitter.images` с `alt` = заголовок статьи).
- **G4. Canonical + OpenGraph/Twitter cards**: `alternates.canonical` в `generateMetadata` для guide-страниц предотвращает duplicate-content-штраф.

## Track H — Content depth (углубление ключевых статей)

Пять статей расширены практическим контентом, которого не хватает в переводе:

- **H1. [Russian LLM в Harness](/guide/russian-llm-harness)** — эталонная `harness.toml` (runnable): multi-provider failover между YandexGPT / GigaChat / локальным vLLM, token budget под кириллицу (+30% к EN-базовой), text-mode fallback для tool-calling, `pdn_filter` для отсева ПДн.
- **H2. [YandexGPT и GigaChat как провайдеры](/guide/yandexgpt-and-gigachat)** — таблица сравнения 6 моделей (YandexGPT 3 / 4 / 4 Pro, GigaChat / MAX / MAX Reasoning) × context window, tool-calling support, типичная latency, сильные/слабые стороны; рекомендации под сценарии.
- **H3. [Open-Source LLM-стек](/guide/open-source-llm-stack)** — `docker-compose.yml` quickstart: vLLM + OpenWebUI на одной машине с GPU deployment через `deploy.reservations.devices`, smoke-test через curl.
- **H4. [Compliance: 152-ФЗ](/guide/compliance-152fz)** — заполненный пример аудита для типового support-бота (категории ПДн, УЗ, согласие, TTL, pseudonymization) + sample journal entry (JSON, хеши вместо payload).
- **H5. [On-Prem Harness](/guide/on-prem-harness)** — ASCII-диаграмма air-gapped deploy: staging → односторонний transfer (data-diode/sneaker-net) → data-сегмент + management-сегмент (read-only для SOC/DPO).
