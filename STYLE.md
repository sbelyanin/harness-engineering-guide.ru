# STYLE.md — конвенция стиля русскоязычного издания

Канон оформления контента. Решает A6 (транслитерация терминов) и A7 (пунктуация/типографика). Дополняет [CONTRIBUTING.md](CONTRIBUTING.md) и раздел «Конвенция перевода» в [ROADMAP.md](ROADMAP.md).

---

## 1. Терминология

### 1.1. Технические термины — на английском

Harness, Agent, Context, Tool, Sandbox, Guardrails, Skill, Sub-Agent, Loop, Memory, Session, Pipeline, Framework, Feature, Feedback, Dashboard, Backend, Frontend, Deploy, Build, Release, Commit, Merge, Branch (как поле/идентификатор).

При первом упоминании можно добавить русский аналог в скобках: «Harness (оболочка)».

### 1.2. Допустимые русизмы

Следующие термины устоялись в русском техническом языке и **остаются кириллицей**:

| Термин | Когда использовать |
|--------|--------------------|
| `баг` | как noun в prose («в логе нашёл баг») |
| `ветка` (git) | только в git-контексте («создай ветку», «удаление веток») |
| `лог` | как noun в prose («смотреть лог», «лог файл» — но `logging` как процесс) |
| `токен` | существительное; `tokenizer`/`tokenize`/`tokens_per_second` — на английском |
| `кэш` / `кэшировать` | как noun/глагол; `cache invalidation` в техническом контексте — английский |
| `лидерборд` / `бенчмарк` | устоявшиеся русизмы для AI-контекстов |

### 1.3. Орфография русизмов (унификация)

| Канон | Запрещён | Примечание |
|-------|----------|------------|
| `хеш` | ~~хэш~~ | как в БТС; «хэш» — устаревающая форма |
| `токен` | ~~токэн~~ | без «э» |
| `кэш` | ~~кеш~~ | через «э» (как в словарях) |
| `бэкенд` / ~~backend~~ | см. § 1.1 — в нашем гайде на английском | — |

## 2. Пунктуация и типографика

### 2.1. Кавычки

- **Только «ёлочки»** для русского текста (U+00AB / U+00BB).
- Допускаются ASCII `"..."` только внутри code-fences, в YAML frontmatter и в инлайн-code.
- Вложенные кавычки — "лапки" (U+201C / U+201D), но в нашем гайде этот случай редок.

### 2.2. Тире

| Символ | Использование |
|--------|---------------|
| `—` (em-dash, U+2014) | тире в прозе: «Harness — это...» |
| `–` (en-dash, U+2013) | диапазоны чисел: «1.2–1.5×», «20–30%», «стр. 10–15» |
| `-` (hyphen) | сложные слова: «CI-пайплайн», «API-вызовы» |

### 2.3. Неразрывные пробелы (NBSP, U+00A0)

Расставляются **автоматически** при рендере HTML — см. `typographRussian()` в `site/lib/content.ts`. Markdown-исходники остаются с обычными пробелами.

Правила:
- После одиночных предлогов и союзов: «и», «а», «но», «в», «с», «к», «у», «о», «об», «на», «по», «до», «из», «от», «за», «для», «или», «что», «как», «не», «ни», «я», «он», «мы», «вы», «это», «же», «ли», «бы».
- Между числом и единицей измерения: «10 кг», «5 минут», «128K токенов».
- После инициалов перед фамилией.

NBSP НЕ ставится:
- Внутри `<code>`, `<pre>`, в атрибутах тегов.
- В code-блоках.

### 2.4. Дефис vs слитно

- Сложные слова через дефис: `CI-пайплайн`, `API-вызовы`, `event-лог`, `e2e-тест`.
- Заимствования — слитно: `subagent` (НЕ `sub-agent` для существительного, но `Sub-Agent` как название роли).
- `как-то`, `где-то`, `когда-либо` — через дефис (русское правило).

## 3. Форматирование

### 3.1. Заголовки

- Каждая статья начинается с `# H1` (то же, что и `title:` в frontmatter).
- Подразделы — `## H2`, далее `### H3`. Уровень `H4+` — редко.
- Code-команды в заголовках — через `code`-format: `## Команда \`git status\``.

### 3.2. Code-блоки

- Всегда с указанием языка: ` ```bash`, ` ```python`, ` ```yaml`.
- Комментарии внутри кода можно переводить, но не сами команды.
- ASCII-диаграммы — в ` ``` ` блоке без языка.

### 3.3. Ссылки

- Внутренние — относительные: `[Guardrails](/guide/guardrails)`.
- Внешние — абсолютные с `target="_blank"` (добавляется автоматически в `content.ts`).
